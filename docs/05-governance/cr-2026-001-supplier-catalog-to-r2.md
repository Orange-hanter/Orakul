  1. КОНТЕКСТ И ПРИЧИНА
═══════════════════════════════════════════════════════════════
Каталог поставщиков и поиск аналогов (US-04, развёрнутые в US-09/10/11 при
последнем обновлении документации 2026-05-21) запланированы на R3 (Q3) как
«Should». Однако по [Business Case §2](../06-financial/01-business-case-and-roi.md)
именно эта функциональность раскрывает Value Driver V2 (оптимизация закупок),
который даёт до 4.2 млн ₽/год на точку — против ~1 млн ₽ от V1.

Без US-04 канал V2 в коммерческом предложении равен 0, и ROI клиента на
60–70% «не виден» в первый год. Кроме того, US-04 — единственная фича со
демонстрируемым ROI на скриншоте до старта пилота, что напрямую влияет на
конверсию Gate G2 (Pilot → Paid ≥80%).

  3. АЛЬТЕРНАТИВЫ И ПОЧЕМУ ОТКЛОНЕНЫ
═══════════════════════════════════════════════════════════════
A) Оставить US-09/10/11 в R3 (как было)
   Плюсы: текущий план не пересматривается, ресурсы фокусируются на
           стабилизации MVP в R2.
   Минусы: коммерческое предложение остаётся без главного value driver
           ещё на квартал; конверсия G2 под риском; конкуренты могут
           подвинуть нас на главном инсайте.
   Почему НЕ выбрали: упущенная экономия для клиента 4.2 млн ₽/год
           недоказана в продаже = клиенты не подписывают paid.

B) Перенести только US-09 (каталог), без US-10/11
   Плюсы: меньше объём работ; каталог — fundament, на котором можно
           показать концепцию.
   Минусы: каталог без поиска аналогов = «справочник», который ни на
           что не влияет; без заявок = clients нет workflow «от
           инсайта к действию».
   Почему НЕ выбрали: фрагментированная ценность; нет демо «вау».

C) Перенести US-09/10/11 в R2 + минимизировать scope (вариант MVP)
   Плюсы: получаем главный value driver в Q2; команда не перегружается
           за счёт сокращения объёма (без импорта xlsx, без рейтингов,
           без полного шаговика заявок — только ручной ввод и
           базовый поиск).
   Минусы: часть функциональности из спецификации придётся переносить
           в R3 как «расширение каталога».
   Почему ВЫБРАЛИ: лучший баланс «время → ценность». См. п. 5.
