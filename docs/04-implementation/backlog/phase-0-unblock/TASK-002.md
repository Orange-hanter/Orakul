## Описание

Получить рабочий API-ключ для тестового окружения Quick Resto, верифицировать базовый доступ. Без рабочего ключа невозможны все последующие спайки (TASK-003, 004, 005).

## Критерии приёмки

- [ ] API-ключ получен и сохранён в секрет-хранилище (или временно — в защищённом канале)
- [ ] Успешный ответ на `GET` любого публичного метода (например, получение списка организаций или меню)
- [ ] Получена документация API (актуальная версия, желательно непубличная расширенная)
- [ ] Подтверждены ограничения sandbox (тестовые данные, rate limits, доступные методы)

## Артефакт

- Запись в секрет-хранилище.
- Ссылка на документацию API сохранена в проекте (будет добавлена в `docs/08-technical/` в Phase 1).

## Риски и блокеры

- Документация API может быть неполной (для коммерческих интеграций) — потребуется уточнение у поддержки.
- Sandbox может не содержать всех методов прод-API — отметить расхождения в TASK-005.
