## Описание

Подтвердить, что рецептуры (нормы закладки ингредиентов в блюдо) доступны через API Quick Resto. Критично для:
- US-02 (Факт vs Норма) — расчёт отклонений невозможен без рецептур
- US-05 (Маржа в реальном времени) — расчёт food cost требует рецептур
- M3 (Экономика меню) в целом

## Критерии приёмки

- [ ] Найден API-метод для получения рецептуры блюда (или зафиксировано его отсутствие)
- [ ] Получен реальный образец данных рецептуры в JSON
- [ ] Описаны доступные поля: ингредиент, количество, единица измерения, потери при ТО (если есть), вариативность по точкам
- [ ] Подтверждено наличие исторической версионности рецептур (если меняются)
- [ ] Результат зафиксирован в [Decision Log](../../../05-governance/05-decision-log.md)

## Риски и блокеры

- **Высокий риск:** если рецептуры недоступны через API, US-02 в текущем виде нереализуема. Альтернативы:
  - Ручной импорт рецептур из бэк-офиса (через выгрузку XLS)
  - Хранение рецептур в Orakul с регулярной сверкой
  - Исключение US-02 из MVP (через CR)
- Этот результат — главный вход в TASK-006 (gate-решение).
