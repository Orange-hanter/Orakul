## Описание

План отката пилота: при каких условиях останавливаем, как возвращаем точку к работе без Orakul, как не повредить операциям клиента.

## Структура документа

1. Триггеры отката:
   - Технические (множественные сбои интеграции)
   - Бизнес (показатели хуже baseline + N%)
   - Операционные (саботаж рекомендаций персоналом)
2. Уровни отката:
   - Soft rollback (отключение конкретного модуля)
   - Full rollback (отключение продукта целиком)
3. Шаги выполнения каждого уровня
4. Кто принимает решение и кто исполняет
5. Связь с [Change Management Plan](../../03-change-management-plan.md)
6. Lessons learned после отката

## Критерии приёмки

- [ ] Триггеры отката определены и измеримы
- [ ] Описаны шаги каждого уровня отката
- [ ] Согласован с операционным директором и пилотным клиентом
- [ ] Документ в статусе `✅ Утверждён`
