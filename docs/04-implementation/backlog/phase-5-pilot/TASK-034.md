## Описание

Определить заведение (или 2), на котором будем проводить пилот. Критерии отбора: качество данных в Kiepper, готовность персонала, бизнес-релевантность, доступность управляющего и собственника.

## Критерии отбора пилотной точки

- [ ] Активная работа на Kiepper не менее 6 месяцев (для исторических данных)
- [ ] Качество данных в Kiepper подтверждено через [Data Audit Checklist](../../04-data-audit-checklist-kiepper.md)
- [ ] Подтверждена готовность собственника / управляющего вкладывать время
- [ ] Бизнес-релевантность: типовой формат, средний размер, ИТ-инфраструктура
- [ ] Географическая близость к команде внедрения (для пилота)

## Критерии приёмки

- [ ] Выбраны 1–2 пилотные заведения с обоснованием
- [ ] Проведён первичный аудит данных по [Data Audit Checklist](../../04-data-audit-checklist-kiepper.md)
- [ ] Подтверждение участия от собственника и управляющего (письменное)
- [ ] Решение зафиксировано в [Decision Log](../../../05-governance/05-decision-log.md)
- [ ] Пилотные клиенты добавлены в [Stakeholder Register](../../../05-governance/02-stakeholder-register.md)

## Артефакт

Запись в [Decision Log](../../../05-governance/05-decision-log.md) + [Stakeholder Register](../../../05-governance/02-stakeholder-register.md).
