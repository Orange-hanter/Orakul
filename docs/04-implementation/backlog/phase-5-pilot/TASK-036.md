## Описание

Зафиксировать измеримые критерии успеха пилота: какие KPI должны достичь каких значений для перехода в Phase 3 — Масштабирование. Без формальных критериев «успех пилота» субъективен.

## Структура раздела

1. Измеряемые KPI (5–7 ключевых из [KPI Framework](../../../02-business/02-kpi-framework-and-dashboard.md))
2. Baseline по каждому KPI (через [Baseline Measurement Protocol](../../07-baseline-measurement-protocol.md))
3. Целевые значения на конец пилота
4. Промежуточные контрольные значения (mid-pilot)
5. Формат sign-off:
   - Кто подписывает «пилот успешен»
   - Кто эскалирует «пилот провален»
   - Источник истины для замеров

## Критерии приёмки

- [ ] Определены 5–7 ключевых KPI пилота
- [ ] Baseline зафиксирован для каждого KPI (или назначен срок замера)
- [ ] Целевые значения согласованы со спонсором и пилотным клиентом
- [ ] Формат sign-off определён
- [ ] Записаны в [Decision Log](../../../05-governance/05-decision-log.md) + раздел в Pilot Plan

## Артефакт

Раздел в [Pilot Plan](TASK-035.md) + запись в [Decision Log](../../../05-governance/05-decision-log.md).
