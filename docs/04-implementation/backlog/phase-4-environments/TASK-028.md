## Описание

Описать окружения: Dev, Staging, Pre-prod (опционально), Prod. Кто имеет доступ, как накатываются миграции, как изолированы данные, как происходит promotion.

## Структура документа

1. Список окружений с назначением
2. Доступы по окружениям (роли и команды)
3. Стратегия данных:
   - Изоляция (тестовые данные vs прод)
   - Маскирование PII в нон-прод
   - Регулярные обновления нон-прод-данных
4. Promotion-процесс (Dev → Staging → Prod)
5. Стратегия миграций БД
6. Стратегия резервного копирования и DR
7. Стратегия секретов по окружениям

## Критерии приёмки

- [ ] Описаны все окружения с назначением
- [ ] Указаны доступы и роли
- [ ] Описана изоляция данных
- [ ] Соответствие [Data Governance](../../../05-governance/03-data-governance.md)
- [ ] Документ в статусе `✅ Утверждён`
