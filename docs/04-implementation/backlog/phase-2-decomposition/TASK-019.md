## Описание

Провести оценочные сессии с командой (planning poker / T-shirt sizing) по всем задачам backlog. Получить story points / часы, скорректировать предварительные оценки из WBS. Без оценок команды Sprint Plan нереалистичен.

## Критерии приёмки

- [ ] Все задачи в трекере имеют оценку команды (story points или часы)
- [ ] Расхождения с предварительной оценкой >50% — обсуждены и обоснованы (либо переоценка, либо декомпозиция)
- [ ] Зафиксирован velocity-prediction (предсказание скорости команды на спринт)
- [ ] Команда подтвердила реалистичность оценок (через ретро-голосование)

## Артефакт

Backlog в трекере с оценками + протокол оценочной сессии в `docs/04-implementation/estimates-session-YYYY-MM-DD.md`.

## Риски и блокеры

- Команда может занизить или завысить оценки. Компенсация — буфер 20% в Sprint Plan.
- Споры по архитектурным решениям во время оценки — заложить тайм-бокс на обсуждение, спорное эскалировать архитектору.
