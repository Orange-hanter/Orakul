## Описание

Разбить каждый User Story из MVP-скоупа на конкретные технические задачи по уровням: backend, frontend, ML, инфра, QA, документация. WBS — мост между TRD и трекером.

## Структура WBS

Иерархическая структура с тремя уровнями:
- **Epic** = модуль PRD (M1, M2, M3, M4)
- **Story** = User Story (US-01, US-02, …)
- **Task** = техническая задача (≤2 чел.-дня)

Каждый task оформляется по [Task Standard](../../templates/task-template.md).

## Критерии приёмки

- [ ] Каждая User Story из MVP разбита на 5–15 задач
- [ ] Все задачи покрывают: бэкенд, фронтенд, ML (где применимо), инфру, QA, документацию
- [ ] Указаны зависимости между задачами
- [ ] Каждая задача имеет предварительную оценку (T-shirt: XS/S/M/L)
- [ ] WBS прошёл review tech-lead и владельцем продукта
- [ ] Документ в статусе `✅ Утверждён`
