## Описание

План релизов: что попадает в Alpha (пилот), что в Beta (расширенный пилот / ранний доступ), что в General Availability. Каждый релиз имеет Exit Criteria и согласован с маркетингом, операциями и пилотным клиентом.

## Структура документа

1. Релизные вехи (Alpha / Beta / GA) с датами
2. По каждому релизу:
   - Цель и аудитория
   - Список включённых User Stories
   - Exit Criteria (что должно быть выполнено для перехода)
   - Risks и план отката
3. Привязка к [Roadmap](../../01-roadmap.md)
4. Coordination с [Communications Plan](../../05-communications-plan.md)
5. Changelog

## Критерии приёмки

- [ ] Определены релизные вехи с датами
- [ ] Каждая User Story MVP привязана к одному из релизов
- [ ] Exit Criteria для каждого релиза определены и измеримы
- [ ] Согласовано с маркетингом, операциями и пилотным клиентом
- [ ] Документ в статусе `✅ Утверждён`
