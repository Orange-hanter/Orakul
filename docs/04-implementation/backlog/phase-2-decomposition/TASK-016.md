## Описание

Утвердить состав команды разработки (внутренняя или подрядная), подписать необходимые договоры, ввести в проект (доступы, ознакомление с документацией). Без команды декомпозиция работ — спекуляция.

## Критерии приёмки

- [ ] Утверждён состав команды:
  - Backend-разработчики (N)
  - Frontend-разработчики (N)
  - ML-инженер (N)
  - QA-инженер (N)
  - DevOps (N)
- [ ] Подписаны договоры (для подрядчиков) или назначены внутренние сотрудники
- [ ] Команда добавлена в [RACI](../../02-raci-matrix.md) и [Stakeholder Register](../../../05-governance/02-stakeholder-register.md)
- [ ] Все члены команды получили доступы к репозиториям, документации, коммуникационным каналам
- [ ] Проведено ознакомление с BRD/PRD/TRD/SAD

## Артефакт

Обновлённый [RACI](../../02-raci-matrix.md) + [Stakeholder Register](../../../05-governance/02-stakeholder-register.md) + подписанные договоры.

## Риски и блокеры

- Сложности найма / выбора подрядчика могут увеличить срок Phase 2. Рекомендуется параллельный поиск с Phase 1.
- Подрядчики с фиксированной ценой потребуют SOW — заложить буфер на его согласование.
