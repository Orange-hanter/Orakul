## Описание

Зафиксировать общекомандные критерии:
- **DoR (Definition of Ready):** когда задача готова к взятию в спринт
- **DoD (Definition of Done):** когда задача завершена и может быть смерджена

Это договорённости внутри команды разработки, дополняющие [Task Standard](../../templates/task-template.md).

## Пример DoR (для иллюстрации)

- [ ] У задачи есть описание и acceptance criteria
- [ ] Есть оценка команды
- [ ] Нет открытых блокеров
- [ ] UX-макеты приложены (для frontend-задач)
- [ ] Тех. дизайн обсуждён (для крупных задач)

## Пример DoD (для иллюстрации)

- [ ] Код написан, ревью пройдено (≥1 одобрение)
- [ ] Unit-тесты добавлены и зелёные
- [ ] Integration-тесты зелёные
- [ ] CI зелёный
- [ ] Документация обновлена
- [ ] Деплой в Staging успешен
- [ ] Acceptance criteria подтверждены QA

## Критерии приёмки

- [ ] DoR описан со списком критериев
- [ ] DoD описан со списком критериев
- [ ] Согласовано командой (включая QA и DevOps)
- [ ] Подписано Tech Lead
- [ ] Документ в статусе `✅ Утверждён`
