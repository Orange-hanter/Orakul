## Описание

Распределить задачи MVP по спринтам (2-недельные, 6–8 спринтов) с учётом зависимостей между задачами, velocity команды и календарных ограничений [Roadmap](../../01-roadmap.md). Каждый спринт получает чёткую цель и набор demo-артефактов.

## Структура каждого спринта

| Поле | Значение |
|------|----------|
| Цель спринта | 1–2 предложения, что должно быть готово |
| Demo-артефакты | Что показываем стейкхолдерам на demo |
| Состав задач | Список из трекера со story points |
| Сумма story points | Соответствие velocity-prediction |
| Зависимости | Внешние блокеры спринта |
| Риски | Что может пойти не так |

## Критерии приёмки

- [ ] Каждый спринт MVP имеет: цель, состав, demo-артефакты, сумму SP
- [ ] Зависимости между задачами не нарушают порядок спринтов
- [ ] Сроки совместимы с [Roadmap](../../01-roadmap.md); расхождения оформлены через CR
- [ ] План согласован с командой (рев) и владельцем продукта (sign-off)
- [ ] Документ в статусе `✅ Утверждён`
