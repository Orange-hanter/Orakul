## Описание

Детальная спецификация интеграции с Quick Resto API: endpoints, форматы запросов/ответов, маппинг полей в каноническую модель Orakul, частоты опроса, обработка ошибок, retry-логика, обработка изменений схемы со стороны Quick Resto.

## Структура документа (рекомендованная)

1. Цели интеграции, скоуп
2. Аутентификация и управление ключами
3. Перечень используемых endpoints (по каждому):
   - URL, метод, параметры
   - Пример запроса
   - Пример ответа
   - Маппинг полей в каноническую модель Orakul
   - Частота опроса (poll-rate)
   - Стратегия дельты (full vs incremental)
4. Обработка ошибок:
   - 4xx (логика на каждый код)
   - 5xx + retry-политика
   - Rate-limit (429) — exponential backoff
5. Обработка stale-data (>2ч из PRD §8.3)
6. Реакция на изменение схемы Quick Resto (диагностика и блокировка)
7. Мониторинг интеграции (метрики, алерты)
8. Changelog

## Критерии приёмки

- [ ] Описаны все используемые endpoints с примерами
- [ ] Каждое поле PRD §8.1 («что читаем») имеет маппинг на конкретное поле API
- [ ] Указаны частоты опроса для каждой сущности (соответствие PRD §8.1)
- [ ] Описаны retry-политики и обработка 4xx/5xx
- [ ] Описана процедура реакции на schema-change
- [ ] Документ согласован с Quick Resto Support (если требуется)
- [ ] Документ в статусе `✅ Утверждён`
