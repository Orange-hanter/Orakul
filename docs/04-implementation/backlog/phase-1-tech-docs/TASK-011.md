## Описание

Описать высокоуровневую архитектуру Orakul: компоненты, потоки данных, технологический стек, схема развёртывания, интеграции, ML-слой, ETL, хранилище, фронт. SAD — точка отсчёта для всех инженерных решений.

## Структура документа (рекомендованная)

1. Цели, контекст
2. Контекстная диаграмма (C4-L1) — Orakul и его окружение
3. Контейнерная диаграмма (C4-L2) — основные сервисы
4. Описание компонентов:
   - ETL-слой (приём данных из Kiepper)
   - Каноническое хранилище
   - ML-сервис (прогнозы, рекомендации)
   - API-шлюз
   - Фронтенд (web + mobile)
   - Сервис аутентификации и RBAC
   - Аудит-лог
5. Технологический стек с обоснованием выбора
6. Схема развёртывания (cloud / on-prem / гибрид)
7. Масштабируемость и отказоустойчивость (SLA 99.5%)
8. Стратегия миграций и эволюции схемы
9. Changelog

## Критерии приёмки

- [ ] Диаграммы C4-L1 и C4-L2 присутствуют
- [ ] Описаны все компоненты
- [ ] Технологический стек зафиксирован: язык бэка, БД, ML-фреймворк, фронтенд-стек, очереди/брокеры
- [ ] Описана схема развёртывания и SLA-инструменты (репликация, бэкап, failover)
- [ ] Утверждено архитектурным комитетом / Tech Lead
- [ ] Документ в статусе `✅ Утверждён`
