## Описание

В [PRD §12.2](../../../03-product/01-product-requirements-document.md#122-открытые-вопросы-на-момент-v10) есть три открытых вопроса, которые блокируют написание TRD и SAD:

1. **LLM vs шаблонизация** для генерации обоснований рекомендаций
2. **Офлайн-режим** для мобильного приложения управляющего
3. **Тариф «только аудит»** (read-only без рекомендаций)

Принять решения по каждому вопросу — иначе TRD будет содержать противоречия.

## Критерии приёмки

- [ ] По каждому из трёх вопросов есть отдельная запись в [Decision Log](../../../05-governance/05-decision-log.md)
- [ ] При выборе LLM — указан конкретный движок, провайдер, fallback-сценарий
- [ ] При выборе шаблонизации — описан подход к шаблонам и их версионированию
- [ ] При решении по офлайн-режиму — определён скоуп (что работает, что нет)
- [ ] При решении по тарифу «только аудит» — определена ценовая модель и срок запуска
- [ ] PRD §12.2 обновлён со ссылками на записи в Decision Log
- [ ] Заинтересованные обновлены ([Communications Plan](../../05-communications-plan.md))

## Артефакт

3 записи в [Decision Log](../../../05-governance/05-decision-log.md) + обновлённый [PRD](../../../03-product/01-product-requirements-document.md).
