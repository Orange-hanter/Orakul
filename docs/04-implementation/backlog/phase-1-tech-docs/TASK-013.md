## Описание

Каноническая модель данных Orakul: сущности, атрибуты, связи, типы, ограничения, версионирование схемы. Data Contract — общий словарь между бэком, ML-слоем и фронтом; основа для миграций и обратной совместимости.

## Структура документа (рекомендованная)

1. Цели и принципы (single source of truth, evolvability)
2. Перечень ключевых сущностей:
   - `SKU` (номенклатура)
   - `Recipe` (рецептура)
   - `Sale` (продажа)
   - `Stock` (остаток)
   - `Movement` (движение)
   - `Writeoff` (списание)
   - `PurchaseOrder` (заказ поставщику)
   - `Supplier` (поставщик)
   - `Price` (цена)
   - `Recommendation` (рекомендация)
   - `AuditEvent` (событие аудита)
3. Для каждой сущности:
   - JSON Schema (или эквивалент)
   - Описания полей с типами и ограничениями
   - Источник данных (Kiepper / собственный / производный)
   - Версионирование
4. Связи между сущностями (ER-диаграмма)
5. Стратегия миграций схемы (semver или эквивалент)
6. Соответствие [Data Governance](../../../05-governance/03-data-governance.md)
7. Changelog

## Критерии приёмки

- [ ] Описаны все ключевые сущности
- [ ] Каждая сущность имеет схему и описание полей
- [ ] Указан источник данных для каждого поля
- [ ] Описано версионирование и стратегия миграций
- [ ] Соответствие [Data Governance](../../../05-governance/03-data-governance.md) подтверждено
- [ ] Документ в статусе `✅ Утверждён`
