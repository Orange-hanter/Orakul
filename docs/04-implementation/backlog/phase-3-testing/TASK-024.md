## Описание

Детальный план тестирования интеграции с Quick Resto API: позитивные сценарии по каждому endpoint, негативные (Kiepper недоступен, схема изменилась, rate limit достигнут), stale-data, нагрузочные.

## Структура документа

1. Цели и скоуп интеграционного тестирования
2. По каждому endpoint:
   - Позитивные сценарии (с примерами запросов)
   - Негативные (4xx, 5xx, timeout)
   - Edge cases (пустой ответ, очень большой ответ, специфические даты)
3. Тестирование stale-data (поведение продукта при возрасте данных >2ч)
4. Тестирование изменений схемы (mock API возвращает изменённую структуру)
5. Mock-сервер Quick Resto: дизайн и набор фикстур
6. Стратегия contract-тестирования (Pact или эквивалент)
7. Тест-данные и их обновление

## Критерии приёмки

- [ ] Test Plan покрывает все endpoints из [Integration Spec](../phase-1-tech-docs/TASK-012.md)
- [ ] По каждому endpoint: позитивные, негативные, edge cases
- [ ] Сценарии для NFR (15-мин latency, 99.5% uptime) описаны
- [ ] Mock-сервер для Quick Resto API спроектирован и описан
- [ ] Документ в статусе `✅ Утверждён`
