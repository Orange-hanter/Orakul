## Описание

Высокоуровневая стратегия тестирования: уровни (unit, integration, contract, E2E, UAT, performance), инструменты, ответственные за каждый уровень, метрики качества (test coverage, bug escape rate), CI-интеграция.

## Структура документа

1. Цели тестирования и принципы
2. Уровни тестирования и их доли в общем покрытии (test pyramid)
3. Инструменты по каждому уровню
4. Ответственные (кто пишет, кто проверяет)
5. Метрики качества и целевые значения
6. Интеграция тестирования в CI/CD
7. Стратегия test data management
8. Стратегия мoking / stubbing внешних систем (Quick Resto, LLM)
9. Bug triage процесс
10. Changelog

## Критерии приёмки

- [ ] Описаны все уровни тестирования и их доли
- [ ] Указаны конкретные инструменты (фреймворки, CI-интеграция)
- [ ] Описаны метрики качества с целевыми значениями
- [ ] Согласовано с tech-lead и владельцем продукта
- [ ] Документ в статусе `✅ Утверждён`
