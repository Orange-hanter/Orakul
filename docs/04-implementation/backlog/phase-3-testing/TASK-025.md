## Описание

План приёмочного тестирования с пилотным клиентом по каждой персоне ([PRD §4](../../../03-product/01-product-requirements-document.md#4-целевые-пользователи-персоны)): сценарии на основе [BDD](../../../03-product/04-bdd-acceptance-scenarios.md) и [Journey Maps](../../../03-product/03-user-journey-maps.md), скрипты, форма sign-off, метрики приёмки.

## Структура документа

1. Цели UAT
2. Состав участников от заказчика (по персонам)
3. Сценарии UAT (с привязкой к BDD)
4. Окружение для UAT (Staging / Pilot)
5. Расписание UAT-сессий
6. Форма sign-off и критерии прохождения
7. Процедура работы с замечаниями

## Критерии приёмки

- [ ] UAT сценарии покрывают все BDD-сценарии MVP
- [ ] Указаны представители заказчика, проводящие UAT, по каждой персоне
- [ ] Форма sign-off и критерии прохождения определены
- [ ] План согласован с владельцем пилота
- [ ] Документ в статусе `✅ Утверждён`
