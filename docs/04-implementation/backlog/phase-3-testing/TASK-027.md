## Описание

План нагрузочного и стресс-тестирования: профили нагрузки, цели по latency/throughput, инструменты, интеграция в CI.

## Структура документа

1. Цели performance-тестирования
2. Профили нагрузки:
   - Типичная (нормальный режим)
   - Пиковая (утро, обеденный час)
   - Стресс (множитель пика)
   - Soak (длительная нагрузка)
3. Сценарии по каждому профилю
4. KPI и SLO:
   - 15-мин latency для рекомендаций ([BRD §7](../../../02-business/01-business-requirements-document.md#7-нефункциональные-требования-бизнес-контекст))
   - 99.5% uptime
   - Открытие экрана ≤2 сек на 4G ([PRD §10](../../../03-product/01-product-requirements-document.md#10-нефункциональные-требования-продуктовый-уровень))
5. Инструменты (k6 / JMeter / Locust)
6. Интеграция в CI/CD (smoke perf на каждый PR + полный прогон на каждый релиз)

## Критерии приёмки

- [ ] Профили нагрузки определены и параметризованы
- [ ] Цели по KPI выписаны и привязаны к BRD/PRD
- [ ] Инструменты выбраны и совместимы с CI
- [ ] Документ в статусе `✅ Утверждён`
