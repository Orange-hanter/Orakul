# RACI Matrix

## 0. Обозначения

| Буква | Значение | Что значит на практике |
|-------|----------|-------------------------|
| **R** — Responsible | Исполнитель | Делает работу |
| **A** — Accountable | Ответственный | Отвечает за результат, подписывает; **A — всегда один** |
| **C** — Consulted | Консультируется | Двусторонняя связь, мнение учитывается |
| **I** — Informed | Информируется | Односторонняя связь, узнаёт о результате |

> Главное правило: **A — ровно один человек на строку.** Если их два — они не отвечают, отвечает «между» ними.

## 6. Эскалационная матрица

| Уровень | Что эскалируется | Решает | SLA |
|---------|------------------|--------|-----|
| L1 | Операционный вопрос внедрения | CSM + PMO | 1 рабочий день |
| L2 | Срыв этапа, отклонение от плана >7 дней | PMO + PO | 2 рабочих дня |
| L3 | Срыв gating-критерия (G1..G4) | Управляющий комитет | внеочередное собрание ≤5 рабочих дней |
| L4 | Стратегическое решение / изменение Vision | CEO + Спонсор | как требует решение |

## 7. Антипаттерны и как их ловить

| Антипаттерн | Как видно | Что делать |
|-------------|-----------|------------|
| Слишком много C | Решения тонут в согласованиях | Оставить C только тех, чьё мнение реально нужно для качества решения |
| Слишком много I | Информационный шум | Объединить в дайджест, использовать [Communications Plan](05-communications-plan.md) |
| Два A | Никто реально не отвечает | Выбрать одного; у второго оставить C |
| A без R на той же стороне | Спонсор «отвечает», но никто «не делает» | Назначить R |
| R без A | «Делает, но никто не подписывает» | Назначить A |

## 8. Регламент изменений RACI

1. Любое изменение RACI оформляется через [Change Request](../05-governance/04-change-request-template.md).
2. Изменение утверждает спонсор + операционный директор клиента.
3. Об изменении уведомляется весь управляющий комитет ≤2 рабочих дня.
4. Версионирование документа — в Changelog ниже.
