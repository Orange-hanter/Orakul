# Design audit — manual UI walkthrough 2026-05-22

Прогон через Playwright по 7 вкладкам и ключевым модалкам на двух viewport-ах: **iPad portrait (820×1180)** и **iPhone (375×812)**. Логин → каждая вкладка → ключевые формы.

Screenshots сохранены в `.playwright-mcp/audit-*.png`.

## 🔴 P0 — критичный системный баг

### A0 `.btn { width: 100% }` ломает все flex-ряды с несколькими кнопками

**Файл:** [app/client/src/styles.css:350-364](../../app/client/src/styles.css)

Декларация:
```css
.btn {
  display: flex;
  ...
  width: 100%;   /* ← виновник */
}
```

**Эффект:** при двух и более `.btn` в одном flex-ряду каждая претендует на 100% ширины. Flex shrinks их равномерно, **выдавливая до 0 любой соседний flex:1 контейнер**. Конкретные сломанные места:

| ID | Где | Что видно пользователю | Скриншот |
|----|-----|-----------------------|----------|
| A1 | **SuppliersTab → детальный экран поставщика** (шапка с «← Назад» / имя / 📥 / ✎) | Имя поставщика **невидимо**, «1 позиция · Активен» переломан вертикально по символу | audit-07, 07b, 24 |
| A19 | **FinanceTab → период-селектор** (select + «⚙ Доходы / расходы») | Select периода схлопнут до chevron, текста периода нет | audit-14, 23 |
| A25 | **DataTab → Интеграции → строка iiko** (icon / name+desc / «Настроить →») | Описание плагина рендерится **по одному слову на строку** | audit-18, 19 |
| A27 | **PluginShell → ряд «Сохранить / Удалить»** | «Сохранить» крошечная (~70px), «Удалить» огромная | audit-20 |
| A28 | **VenueSelector → строка точки** (radio / имя+адрес / ✎) | Имя «Точка 1» переломано на 2 строки | audit-21 |

Все 5 проявлений — одно и то же. Фикс в одном месте.

**Места, где `.btn` корректно занимает 100% и нужно сохранить поведение:**
- Login → кнопка «Войти»
- Modal.jsx footer → «Отмена» (один или с другими `.modal-footer .btn` → `flex: 1`)
- Telegram → «Подключить»
- Резервная копия → «Скачать .enc» / «Загрузить»
- Order wizard → «Далее →» / «← Назад»

**Рекомендованный фикс:**

```css
/* В .btn убрать width: 100% — default ширина по контенту */
.btn { /* без width */ }

/* Добавить модификатор для случаев full-width */
.btn-block { width: 100%; }
```

Затем в местах, где кнопка должна тянуться на всю ширину, добавить класс `btn-block`. Список таких мест по grep:
- `Login.jsx` — кнопка submit
- `Modal.jsx` — кнопки в `.modal-footer` (уже есть `.modal-footer .btn { flex:1 }` — этого достаточно)
- `DataTab.jsx` — «Подключить» Telegram, «Скачать», «Загрузить»
- `MenuTab.jsx` — «+ Добавить ингредиент», «Удалить блюдо»
- `OrderWizard` — все 3 (внутри Modal footer уже работают через `.modal-footer .btn`)
- `SuppliersTab` — `+ Добавить поставщика` и подобные одиночные

**Альтернатива (более ленивая):** оставить `.btn { width: 100% }`, в каждом сломанном месте добавить inline `style={{ width: 'auto' }}`. ~6 точечных правок. Не рекомендую — баг повторится в новом коде.

## 🟡 P1 — функциональные

### A18 «Валовая маржа» KPI карточка — двойной BYN

**Файл:** [FinanceTab.jsx](../../app/client/src/components/tabs/FinanceTab.jsx)

`sub={`${fmtMoney(grossProfit)} BYN`}` — но `fmtMoney` уже добавляет ` BYN` в конце. Результат: «26 BYN BYN».

**Фикс:** `sub={fmtMoney(grossProfit)}` (убрать ручную добавку).

### A14 OrderWizard — «1 позиций» вместо «1 позиция»

**Файл:** [OrdersTab.jsx](../../app/client/src/components/tabs/OrdersTab.jsx)

```jsx
{(itemsBySupplier.get(s.id) || []).length} позиций
```

**Фикс:** русские правила множественного числа. Помощник:
```js
function plural(n, [one, few, many]) {
  const m10 = n % 10, m100 = n % 100;
  if (m10 === 1 && m100 !== 11) return one;
  if (m10 >= 2 && m10 <= 4 && (m100 < 10 || m100 >= 20)) return few;
  return many;
}
// plural(n, ['позиция', 'позиции', 'позиций'])
```

Применить везде, где есть числовые лейблы:
- «N позиций» в OrderWizard + SuppliersTab + DataTab
- «N блюд» в DataTab «24 блюд» (тоже неверно для N=1/2-4)

## 🟡 P2 — UX

### A17 Кнопка «Далее →» не отключается визуально когда нельзя продвинуться

**Файл:** [OrdersTab.jsx](../../app/client/src/components/tabs/OrdersTab.jsx)

В step 2 с qty=0 onSave проверяет `canNext` и просто не вызывает setStep. Кнопка выглядит активной (blue) — пользователь жмёт, ничего не происходит.

**Фикс:** добавить `saving={!canNext || saving}` в Modal или прокинуть `disabled` отдельным prop.

### A6 «Удалить блюдо» в edit-форме слишком доминирует визуально

**Файл:** [MenuTab.jsx](../../app/client/src/components/tabs/MenuTab.jsx)

Красная full-width кнопка между рецептом и footer-ом приковывает внимание сильнее «Сохранить». Психологически инверсия приоритетов.

**Фикс:** сделать иконкой `🗑 Удалить` рядом с заголовком модалки, или ghost-стиль с красным текстом без фона.

- Settings modal (Финансы) — список выручки + расходов, edit/delete рядом
