# Спецификация модуля «Финансовая аналитика»

## 1. Назначение

Модуль агрегирует данные из всех источников (QuickResto, Kiepper, заявки поставщикам) и предоставляет финансовому контролёру, управляющему и собственнику единую аналитическую картину: P&L, себестоимость, маржинальность, закупочные затраты, отклонения от плана.

**Периоды:** день, неделя, месяц, квартал, произвольный диапазон.

**Детализация:** клик на любую строку разворачивает подстроки до уровня категории / блюда / дня.

## 6. Аналитика меню (Food Cost)

### 6.1 Таблица маржинальности блюд

```
┌────────────────────────────────────────────────────────────────────┐
│  Меню   Фильтр: [Все категории ▾]  Сортировка: [Food Cost % ▾]    │
│  ─────────────────────────────────────────────────────────────────│
│  Блюдо               Цена    Себест.  FC %   Маржа BYN  Тренд     │
│  ─────────────────────────────────────────────────────────────────│
│  🔴 Стейк из лосося  28.00   14.20   50.7%   13.80  BYN  ▲ +2%FC  │
│  🟡 Боул с тунцом    18.50    7.40   40.0%   11.10  BYN  — 0%     │
│  🟢 Капучино         5.50     0.90   16.4%    4.60  BYN  ▼ -1%FC  │
│  🟢 Круассан         4.20     1.05   25.0%    3.15  BYN  — 0%     │
│                                                                    │
│  🔴 FC > 40%  🟡 FC 30–40%  🟢 FC < 30%                           │
└────────────────────────────────────────────────────────────────────┘
```

**Drill-down по блюду:** клик открывает детализацию до ингредиентов с текущими ценами и нормами закладки.

**Обновление в реальном времени:** при изменении цены поставщика в каталоге FC % пересчитывается автоматически и подсвечивается изменение.

### 6.2 ABC-анализ меню

```
A — высокий оборот + высокая маржа  → «Звёзды» (поддерживать)
B — высокий оборот + низкая маржа   → «Рабочие лошадки» (оптимизировать)
C — низкий оборот + высокая маржа   → «Загадки» (продвигать)
D — низкий оборот + низкая маржа    → «Собаки» (рассмотреть вывод)
```

Матрица отображается как 2×2 scatter-plot по осям «количество продаж» и «маржа BYN».

## 8. KPI-дашборд

### 8.1 Виджеты главного экрана

| Виджет | Метрика | Норма | Алёрт |
|--------|---------|-------|-------|
| Food Cost % | Процент себестоимости от выручки | ≤ 32% | > 38% |
| Waste % | Списания сверх нормы / выручка | ≤ 2% | > 4% |
| Gross Margin % | Валовая маржа | ≥ 65% | < 58% |
| EBITDA % | Операционная прибыль | ≥ 25% | < 15% |
| Закупки vs план | Отклонение фактических закупок | ≤ 5% | > 15% |
| AI Acceptance Rate | % принятых рекомендаций | ≥ 60% | < 40% |

### 8.2 Настройка норм

Каждый порог настраивается на уровне организации и точки:

```
Настройки → KPI → Нормы

  Food Cost %        Целевое: [32%]  Алёрт при: [38%]
  Gross Margin %     Целевое: [65%]  Алёрт при: [58%]
  EBITDA %           Целевое: [25%]  Алёрт при: [15%]
  [Сохранить]
```

## 10. Модель данных

```sql
-- Агрегированные дневные показатели точки
CREATE TABLE venue_daily_financials (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    venue_id        UUID NOT NULL REFERENCES venues(id),
    date            DATE NOT NULL,
    revenue_pos     NUMERIC(14,2),    -- выручка из QuickResto
    revenue_other   NUMERIC(14,2),    -- прочая (ручной ввод)
    cogs_purchases  NUMERIC(14,2),    -- закупки (из заявок)
    cogs_waste      NUMERIC(14,2),    -- сверхнормативные списания
    gross_profit    NUMERIC(14,2) GENERATED ALWAYS AS
                      (revenue_pos + revenue_other - cogs_purchases - cogs_waste) STORED,
    food_cost_pct   NUMERIC(5,2),     -- рассчитывается ETL-джобом
    fixed_costs     NUMERIC(14,2),    -- из таблицы fixed_expenses
    ebitda          NUMERIC(14,2),
    updated_at      TIMESTAMPTZ DEFAULT now(),
    UNIQUE (venue_id, date)
);

-- Постоянные расходы (ручной ввод)
CREATE TABLE fixed_expenses (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    venue_id        UUID NOT NULL REFERENCES venues(id),
    name            TEXT NOT NULL,
    amount          NUMERIC(14,2) NOT NULL,
    currency        TEXT DEFAULT 'BYN',
    category        TEXT,            -- 'rent','payroll','utilities','other'
    recurrence      TEXT,            -- 'monthly','quarterly','one-time'
    start_date      DATE NOT NULL,
    end_date        DATE,
    created_by      UUID REFERENCES users(id)
);

-- Плановые показатели
CREATE TABLE financial_targets (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    venue_id        UUID,            -- null = уровень организации
    org_id          UUID NOT NULL,
    period_start    DATE NOT NULL,
    period_end      DATE NOT NULL,
    metric          TEXT NOT NULL,   -- 'revenue','food_cost_pct','ebitda_pct'
    target_value    NUMERIC(14,4),
    alert_threshold NUMERIC(14,4),
    created_by      UUID REFERENCES users(id)
);
```

## 12. Критерии приёмки

- [ ] P&L за месяц загружается за ≤ 3 секунды при объёме ≤ 5 000 заказов.
- [ ] Food Cost % для блюда пересчитывается автоматически при обновлении цены поставщика; задержка ≤ 15 мин.
- [ ] Ручной ввод постоянных расходов с автоповтором «Ежемесячно» создаёт записи на 12 месяцев вперёд при сохранении.
- [ ] Пользователь без роли `fin_controller`, `manager`, `ops_director` или `owner` не видит раздел P&L и получает 403.
- [ ] Экспорт P&L в PDF содержит корректные итоги и не имеет обрезанных строк.
- [ ] Автоматическая рассылка ежедневного среза отправляется в ±5 минут от заданного времени.
- [ ] ABC-анализ меню корректно строится при наличии ≥ 10 блюд; при < 10 — показывает предупреждение «недостаточно данных».
- [ ] Нормы KPI (Food Cost %, EBITDA %) сохраняются на уровне точки и переопределяют значения организации.
