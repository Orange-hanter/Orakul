# Спецификация модуля «Каталог поставщиков»

> ⚠️ **Scope (по состоянию на Sprint 2026-05-22)**
>
> Эта спецификация описывает **полный целевой объём** модуля (R3+). На проде сейчас работает **MVP-подмножество**.
>
> | Возможность | MVP (R2, в проде) | Full (R3+, отложено) |
> |---|:---:|:---:|
> | Список поставщиков + ассортимент | ✅ | ✅ |
> | Ручной ввод цены | ✅ | ✅ |
> | История цен (автозапись на изменение) | ✅ | ✅ |
> | Поиск аналогов: точное совпадение по productId | ✅ | ✅ |
> | Поиск аналогов: fuzzy по имени (русский стеммер + Jaccard) | ✅ | ✅ |
> | Импорт прайсов из .xlsx / .csv с маппингом колонок | ✅ | ✅ |
> | Импорт прайсов из .pdf через OCR | ❌ | ✅ |
> | Авто-обновление существующих позиций при импорте (вместо дублирования) | ❌ | ✅ |
> | Рейтинг поставщиков (скользящее среднее) | ❌ | ✅ |
> | Нормализация цен по единицам (1 л vs 250 мл) | ❌ | ✅ |
> | Деактивация поставщика с блокировкой новых заявок | ❌ | ✅ |
> | API эндпоинты в виде `/api/v1/suppliers` (REST) | ❌ записи через общий `/api/records` | ✅ |

## 1. Назначение

Модуль предоставляет закупщику и управляющему **единое место** для работы с поставщиками: просмотр ассортимента, мониторинг актуальных цен, анализ истории изменений и автоматический поиск более выгодных аналогов у конкурирующих поставщиков.

## 3. Функциональные блоки

### 3.1 Список поставщиков

| Поле | Описание |
|------|----------|
| Название | Юридическое и торговое наименование |
| Рейтинг надёжности | 1–5 звёзд, рассчитывается на основе истории: % просрочек, % брака, % несоответствия заявке |
| Статус | `активный`, `приостановлен`, `на согласовании` |
| Последняя сделка | Дата последнего оформленного заказа |
| Контактное лицо | Имя, телефон, e-mail |
| Теги | `мясо`, `молочка`, `бакалея`, `алкоголь` — для фильтрации |

**Сортировка по умолчанию:** активные → по рейтингу → алфавит.

### 3.2 Ассортимент поставщика

Каждый поставщик имеет таблицу позиций:

| Поле | Тип | Описание |
|------|-----|----------|
| `supplier_id` | UUID | FK к таблице поставщиков |
| `item_id` | UUID | FK к внутреннему справочнику ингредиентов |
| `supplier_sku` | string | Артикул поставщика |
| `item_name_supplier` | string | Название у поставщика (может отличаться) |
| `unit` | enum | кг, л, шт, упак |
| `price_current` | decimal | Актуальная цена |
| `currency` | BYN/RUB/USD |  |
| `min_order_qty` | decimal | Минимальная партия |
| `delivery_days` | int | Среднее время доставки (рабочих дней) |
| `vat_rate` | decimal | НДС % |
| `valid_from` | date | Дата начала действия цены |
| `valid_to` | date | Дата окончания (null = бессрочно) |

**Импорт прайс-листов:** поддерживаются форматы `.xlsx`, `.xls`, `.csv` (реализовано в R2 MVP, см. [Import Flow](#import-flow)). Формат `.pdf` через OCR-парсер — отложено на R3. Ручной ввод через форму.

#### Import Flow {#import-flow}

Кнопка **📥 Импорт прайс-листа** в карточке поставщика. UI-поток:

```
[Выбор файла] → [Авто-детекция колонок] → [Превью + ручной маппинг] → [Импорт]
```

**Авто-детекция:** заголовки сопоставляются с полями по ключевым словам (название / цена / единица / мин. партия / срок доставки / артикул). Пользователь может скорректировать маппинг вручную.

**Авто-привязка к товару склада:** если `itemName` точно совпадает с именем существующего `product` (case-insensitive, trim) — `productId` проставляется автоматически. Fuzzy-матчинг — в R3.

**Валидация строк:** обязательны `itemName` + `price`. Невалидные строки подсвечиваются в превью и пропускаются при импорте.

**Производительность:** xlsx-парсер (~163 KB gzip) загружается lazy-чанком только при открытии модалки. Создание позиций — последовательные API-вызовы; для 100 позиций ~5–10 сек.

### 3.3 История цен

При клике на строку позиции — боковая панель с графиком и таблицей изменений:

```
┌─────────────────────────────────────────────┐
│ История цен: Мука в/с                        │
│ Поставщик: ООО АгроПоставка                 │
│ ─────────────────────────────────────────── │
│  BYN/кг                                     │
│  1.30 ┤                  ╭─                 │
│  1.25 ┤          ╭───────╯                  │
│  1.20 ┤    ╭─────╯        ← текущая          │
│  1.15 ┤────╯                                │
│       └──────────────────────────────────── │
│       янв   фев   мар   апр   май            │
│                                             │
│  Дата         Цена    Изменение    Источник  │
│  2026-05-01   1.20 BYN  ▲ +4%    прайс v12  │
│  2026-03-15   1.15 BYN  ▲ +4.5%  прайс v11  │
│  2026-01-10   1.10 BYN  —        прайс v09  │
└─────────────────────────────────────────────┘
```

**Хранение истории:** все изменения цены сохраняются с датой, источником (импорт/ручной ввод) и пользователем.

## 4. Поиск похожих товаров у других поставщиков

### 4.1 Триггер

Кнопка **«Найти аналоги дешевле»** доступна для каждой строки позиции в ассортименте поставщика, а также в виде глобального запроса по всему ассортименту текущего поставщика.

### 4.2 Алгоритм матчинга

```
1. Нормализация имени позиции:
   - приведение к нижнему регистру
   - удаление единиц измерения из названия
   - стемминг (мука → мук, масло → масл)

2. Поиск по справочнику ингредиентов:
   - точное совпадение по item_id (если артикул уже привязан)
   - нечёткий поиск (Levenshtein distance ≤ 2) по нормализованному имени
   - фильтр по категории ингредиента

3. Ранжирование результатов:
   - основной критерий: цена на единицу (с пересчётом в единую ЕИ)
   - вторичный: рейтинг поставщика
   - третичный: срок доставки
```

### 4.3 Интерфейс результатов

```
┌──────────────────────────────────────────────────────────────────┐
│ Аналоги для: Мука в/с   (текущая цена: 1.20 BYN/кг)             │
├──────────────────────────────────────────────────────────────────┤
│ Поставщик          │ Позиция         │ Цена   │ ∆ к текущей │ ★  │
├────────────────────┼─────────────────┼────────┼─────────────┼────┤
│ ИП Козлов          │ Мука пшен. в/с  │ 1.05   │ ▼ -12.5%   │4.5 │
│ ООО БелФрут        │ Мука высш. сорт │ 1.10   │ ▼ -8.3%    │3.8 │
│ ООО ФрешМаркет     │ Мука в/с ГОСТ   │ 1.18   │ ▼ -1.7%    │4.1 │
├────────────────────┴─────────────────┴────────┴─────────────┴────┤
│   [Сравнить выбранных]              [Переключить поставщика]     │
└──────────────────────────────────────────────────────────────────┘
```

**Потенциальная экономия** отображается суммарно по всему ассортименту с возможностью фильтра по экономии > X%.

### 4.4 Действие «Переключить поставщика»

Запускает [пошаговый мастер заявки](07-order-request-spec.md#мастер-заявки) с предзаполненными полями:
- позиция из аналога
- новый поставщик
- количество — из последнего заказа у текущего поставщика

## 5. Управление поставщиками (CRUD)

### 5.1 Добавление поставщика

Форма с полями: название, юр. реквизиты (опционально), контактное лицо, категории товаров, условия оплаты, условия доставки. После сохранения — предложение загрузить прайс-лист.

### 5.2 Деактивация поставщика

Перевод в статус `приостановлен`. Уже оформленные активные заявки к этому поставщику — остаются, новые — блокируются с предупреждением.

### 5.3 Оценка поставщика

После каждой принятой поставки (статус заявки → `принято`) пользователю предлагается оценить: соответствие количества, качество, соблюдение сроков. Рейтинг пересчитывается автоматически как скользящее среднее за последние 20 сделок.

---

## 6. Привязка ингредиентов справочника к поставщикам

Каждый ингредиент из внутреннего справочника может быть привязан к одному или нескольким поставщикам:

```
Ингредиент: Масло сливочное 82.5%
├── ООО АгроПоставка   8.50 BYN/кг  [основной]
├── ИП Козлов          8.20 BYN/кг  [резервный]
└── ООО ФрешМаркет     8.90 BYN/кг  [разовый]
```

**Основной поставщик** используется по умолчанию при формировании рекомендаций заказа (US-03). Резервный — при автоматическом поиске замены, если основной недоступен.

## 8. Модель данных (дополнение к [Data Model](02-data-model-and-pipeline.md))

```sql
-- Поставщики
CREATE TABLE suppliers (
    id          UUID PRIMARY KEY,
    org_id      UUID NOT NULL REFERENCES organizations(id),
    name        TEXT NOT NULL,
    legal_name  TEXT,
    status      TEXT CHECK (status IN ('active','paused','pending')),
    rating      NUMERIC(2,1),          -- 1.0–5.0, пересчёт триггером
    tags        TEXT[],
    created_at  TIMESTAMPTZ DEFAULT now()
);

-- Ассортимент поставщика (текущие цены)
CREATE TABLE supplier_catalog (
    id              UUID PRIMARY KEY,
    supplier_id     UUID REFERENCES suppliers(id),
    item_id         UUID REFERENCES ingredients(id),
    supplier_sku    TEXT,
    item_name_raw   TEXT NOT NULL,
    unit            TEXT NOT NULL,
    price_current   NUMERIC(12,4) NOT NULL,
    currency        TEXT DEFAULT 'BYN',
    min_order_qty   NUMERIC(12,3),
    delivery_days   INT,
    vat_rate        NUMERIC(5,2),
    valid_from      DATE,
    valid_to        DATE,
    updated_at      TIMESTAMPTZ DEFAULT now()
);

-- История цен (append-only)
CREATE TABLE supplier_price_history (
    id            UUID PRIMARY KEY,
    catalog_id    UUID REFERENCES supplier_catalog(id),
    price         NUMERIC(12,4) NOT NULL,
    currency      TEXT,
    changed_at    TIMESTAMPTZ DEFAULT now(),
    source        TEXT,           -- 'import','manual','api'
    changed_by    UUID REFERENCES users(id)
);
```

## 10. Критерии приёмки (AC)

- [ ] При загрузке прайс-листа (xlsx) система парсит его и показывает предпросмотр с возможностью маппинга колонок до сохранения.
- [ ] История цен сохраняется при каждом изменении; нельзя удалить запись истории.
- [ ] Поиск аналогов возвращает результаты за ≤ 3 секунды при < 50 поставщиков.
- [ ] Потенциальная экономия рассчитывается с учётом минимальной партии (не «на бумаге», а реально достижимая).
- [ ] Рейтинг поставщика обновляется автоматически при смене статуса заявки на `принято`.
- [ ] Деактивированный поставщик недоступен для новых заявок, но его данные остаются в истории.
