# Спецификация модуля «Формирование заявок на заказ»

## 1. Назначение

Модуль обеспечивает **удобный пошаговый процесс** создания, согласования и отправки заявки на заказ поставщику. Цель — снизить ручной труд закупщика и исключить ошибки при оформлении заказов нескольким партнёрам одновременно.

---

## 2. Точки входа

| Точка входа | Описание |
|-------------|----------|
| Вкладка «Заявки» → «Новая заявка» | Ручной запуск мастера |
| Модуль рекомендаций (US-03) → «Оформить заявку» | Предзаполнение из рекомендации ИИ |
| Каталог поставщиков → «Переключить поставщика» | Предзаполнение из аналогового поиска |
| Алёрт по остаткам (US-01) → «Заказать» | Предзаполнение из алёрта |

---

## 3. Мастер заявки (Step-by-Step Wizard)

Мастер состоит из 5 шагов. Пользователь может вернуться на любой предыдущий шаг. Прогресс сохраняется автоматически как черновик.

```
 ●─────────────────────────────────────────────────────
 Шаг 1        Шаг 2        Шаг 3        Шаг 4    Шаг 5
 Партнёры  →  Позиции   →  Условия  →  Обзор  →  Отправка
 ─────────────────────────────────────────────────────●
```

```
┌──────────────────────────────────────────────────────────────┐
│  Шаг 1 из 5: Выбор поставщиков                               │
│  ─────────────────────────────────────────────────────────── │
│  🔍 Поиск поставщика...                                      │
│                                                              │
│  ☑  ООО АгроПоставка    ★★★★☆  мясо, молочка, бакалея       │
│  ☑  ИП Козлов           ★★★★★  бакалея                      │
│  ☐  ООО ФрешМаркет      ★★★☆☆  овощи, фрукты               │
│  ☐  ООО БелФрут         ★★★★☆  фрукты                       │
│                                                              │
│  ⚠  Выбрано поставщиков: 2                                   │
│  Система автоматически разобьёт позиции по поставщикам.     │
│                                                              │
│                              [Отмена]  [Далее →]            │
└──────────────────────────────────────────────────────────────┘
```

**Правила:**
- Можно выбрать от 1 до N поставщиков за один сеанс.
- При выборе нескольких поставщиков система покажет, какие позиции есть только у одного из них, и предложит распределение.
- Приостановленные поставщики показываются с меткой, добавить нельзя.

### Шаг 2 — Формирование списка позиций

```
┌──────────────────────────────────────────────────────────────────┐
│  Шаг 2 из 5: Позиции заявки                                      │
│  ──────────────────────────────────────────────────────────────  │
│  [Добавить из рекомендаций ИИ]  [Добавить из списка]  [+ Вручную]│
│                                                                  │
│  Позиция          Ед.   Кол-во     Поставщик         Цена за ед. │
│  ─────────────────────────────────────────────────────────────── │
│  Мука в/с         кг    [  50  ] ▾ ООО АгроПоставка   1.20 BYN  │
│                         ↑рекоменд.                 💡 ИП Козлов  │
│                                                       дешевле 12%│
│  Масло слив.82.5% кг    [  10  ] ▾ ООО АгроПоставка   8.50 BYN  │
│  Яйцо С1          шт   [ 100  ] ▾ ИП Козлов            0.43 BYN  │
│  + Добавить позицию                                              │
│  ─────────────────────────────────────────────────────────────── │
│  Итого: 2 поставщика / 3 позиции                                 │
│  Предварительная сумма: ~97.30 BYN                               │
│                              [← Назад]  [Далее →]               │
└──────────────────────────────────────────────────────────────────┘
```

**Функции шага 2:**

| Функция | Описание |
|---------|----------|
| «Добавить из рекомендаций ИИ» | Подтягивает актуальный список рекомендаций модуля US-03 |
| «Добавить из списка» | Поиск по справочнику ингредиентов с фильтром по категориям |
| Ручной ввод | Позиция + количество + поставщик + цена вручную |
| Смена поставщика для позиции | Выпадающий список поставщиков, у которых есть данная позиция |
| Подсказка «дешевле» | Inline-подсказка при наличии аналога по меньшей цене |
| Изменение количества | Поле ввода с валидацией min_order_qty; предупреждение если ниже минимума |

```
│                                                                  │
│  ИП Козлов                                                       │
│    Желаемая дата доставки:  [23 мая 2026    ▾]                   │
│    Адрес доставки:          [Проспект Мира 12, кухня ▾]          │

│                                                                  │

│                                                                  │
│                              [← Назад]  [Далее →]               │

```

**Правила:**
- Дата доставки не может быть раньше сегодня + время доставки поставщика (delivery_days).
- Если выбранный день — выходной для поставщика (если задан в его карточке), система предупреждает.
- Способ передачи заявки хранится как настройка по умолчанию для каждого поставщика.

## 4. Жизненный цикл заявки

```
черновик → на согласовании → подтверждена → отправлена → принята → закрыта
                 ↓
             отклонена → черновик (редактирование)
```

| Статус | Кто может изменить | Описание |
|--------|--------------------|----------|
| `черновик` | purchaser, manager | Создана, не отправлена |
| `на согласовании` | manager, ops_director | Ожидает одобрения по лимиту |
| `подтверждена` | — | Одобрена, ждёт отправки |
| `отправлена` | — | Передана поставщику |
| `частично принята` | purchaser, manager | Поставщик привёз часть позиций |
| `принята` | purchaser, manager | Поставка принята полностью |
| `закрыта` | manager | Завершена, архив |
| `отменена` | manager, ops_director | Отмена до отправки |

| Событие | Получатель | Канал |
|---------|-----------|-------|
| Заявка требует согласования | manager, ops_director | Push + Telegram |
| Заявка подтверждена | purchaser | Push |
| Заявка отклонена | purchaser | Push + Telegram |
| Поставка не пришла в срок | purchaser, manager | Push + Telegram |
| Поставщик подтвердил получение | purchaser | Push |

## 9. Критерии приёмки

- [ ] Мастер заявки сохраняет черновик при каждом переходе между шагами; при случайном закрытии вкладки черновик восстанавливается.
- [ ] Позиции из рекомендаций ИИ подтягиваются за ≤ 2 сек.
- [ ] При выборе количества ниже min_order_qty — предупреждение, но не блокировка.
- [ ] Лимит согласования проверяется на Шаге 4; пользователю показывается имя согласующего.
- [ ] Отправка заявки по email/Telegram происходит асинхронно; статус «доставлено/ошибка» обновляется в интерфейсе.
- [ ] История изменений заявки (кто, что, когда) сохраняется и доступна в детальной карточке.
- [ ] Заявку в статусе `принята` нельзя редактировать; только добавить комментарий.
