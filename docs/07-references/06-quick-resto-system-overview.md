# Quick Resto — обзор системы (Windows-терминал v2.52.1.0)

**Дата:** май 2026  
**Источник:** официальная документация quickresto.ru, myquickresto.ru (release notes)  
**Контекст:** анализ системы с точки зрения внешней интеграции и получения исторических данных кухни.

---

## 1. Идентификация версии 2.52.1.0

Quick Resto имеет **три отдельных кассовых приложения** с независимой нумерацией версий:

| Приложение | Платформа | Формат версии | Контекст на май 2026 |
|---|---|---|---|
| **Quick Resto Касса** | iOS (iPad) | `2.XX` | 2.53 — март 2026 |
| **Quick Resto** | Android | `2.XX` | Отдельный трек |
| **Windows-терминал** | Windows 10 64-bit | `2.XX.X.X` | **2.52.1.0** — исследуемая версия |

**Вывод:** версия `2.52.1.0` с четырёхкомпонентной нумерацией — это **Windows-терминал** Quick Resto. Публичных release notes для Windows-версий на myquickresto.ru нет — они не выходят отдельными страницами, в отличие от iOS-кассы и бэк-офиса.

**Хронологический контекст:**
- Касса iOS 2.51 → BO 2.93–2.94, ноябрь 2025
- Windows-терминал 2.52.1.0 → ориентировочно декабрь 2025 – февраль 2026
- Касса iOS 2.53 → BO 2.98, март 2026

## 2. Архитектура системы

```
┌──────────────────────────────────────────────────────┐
│                  Quick Resto Cloud                   │
│              (бэк-офис, BO 2.99 — апр. 2026)         │
└────────────┬──────────────────────────┬──────────────┘
             │                          │
     ┌───────▼───────┐          ┌───────▼────────┐
     │  iOS Касса    │          │ Windows-терм.  │
     │   v2.53       │          │  v2.52.1.0     │
     │  (iPad)       │          │  (ПК/моноблок) │
     └───────────────┘          └────────────────┘
             │
     ┌───────▼────────┐
     │ Quick Resto    │
     │ Kitchen (KDS)  │
     │ (отдельн. iPad)│
     └────────────────┘
```

Все терминалы работают с единым облачным бэк-офисом. Данные синхронизируются через интернет. Офлайн-режим поддерживается с ограничениями (терминалы не синхронизируются между собой без интернета).

## 3. Windows-терминал v2.52.1.0 — функциональность

Позиционируется как версия для **малого общепита в формате быстрого обслуживания**: кофейня, пиццерия, фудтрак, фастфуд. Это **урезанная версия** по сравнению с iOS-кассой.

### 3.1. Поддерживаемые операции

| Функция | Наличие |
|---|---|
| Быстрые заказы (без стола) | Да |
| Заказы на стол | Да |
| Модификаторы к блюдам | Да |
| Скидки и акции | Да |
| Онлайн-заказы и предзаказы | Да |
| Отложенные заказы | Да |
| Авторизация гостя, списание и начисление бонусов | Да |
| Стоп-лист (добавление блюд/модификаторов) | Да |
| Печать чека на кухню | Да |
| Печать пречека | Да |
| Печать чека курьеру | Да |
| Просмотр чеков за текущую смену | Да |
| Возврат чека (только текущая смена) | Да |
| Инкассация | Да |
| Схема зала / управление посадкой | **Нет** |
| Межтерминальная синхронизация без интернета | **Нет** |
| Полная аналитика кухни | **Нет** |

### 3.2. Системные требования

- **ОС:** Windows 10 64-bit (чистая установка)
- **Важно:** не должно быть старых драйверов фискальных регистраторов и сторонних приложений
- **Форм-факторы:** компьютер, моноблок, ноутбук, планшет на Windows

## 5. API для внешней интеграции

| Параметр | Значение |
|---|---|
| Тип | REST/JSON |
| Документация | quickresto.ru/api |
| Публичный API с | 14 марта 2016 г. |
| Аутентификация | API Key |
| Готовые коннекторы | Albato (800+ сервисов), ApiMonster |
| Webhook | Да |

### 5.1. Доступные данные через API

| Тип данных | Доступность |
|---|---|
| История транзакций (заказы) | Да |
| Позиции заказа (блюда, количество, цена) | Да |
| Остатки ингредиентов на складе | Да |
| История движения ингредиентов | Да |
| Текущий стоп-лист | Да |
| Финансовая статистика за период | Да |
| Номенклатура (меню) | Да |
| Отмены/удаления позиций (отказы) | **Не задокументировано** |
| KDS-тайминги (время приготовления) | **Нет** |

### 5.2. Ограничения API относительно конкурентов

- Нет аналога OLAP-отчётности iiko — аналитика ограничена стандартными отчётами
- Отмены и удаления блюд явно не выделены как отдельный тип события в API
- KDS-тайминги не отдаются через API ни в каком виде
- Документация значительно беднее, чем у iiko (нет публичного SDK, нет Postman-коллекций)

## 6. Модуль KDS — Quick Resto Kitchen

Quick Resto Kitchen — **отдельное iOS-приложение** для кухонного экрана:
- Устанавливается на отдельный iPad
- Отображает очередь заказов в реальном времени
- Поддерживает несколько экранов на одну кухонную станцию (синхронизация статусов между экранами)
- **Исторические данные таймингов через API не доступны** — нет механизма экспорта событий KDS

## 9. Сильные и слабые стороны

### Сильные стороны
- Низкий порог входа по цене
- Облако без необходимости локального сервера
- Работает на любом Windows 10 устройстве (в т.ч. дешёвые планшеты)
- Офлайн-режим с автосинхронизацией
- Широкие интеграции через агрегаторы и Albato

### Слабые стороны
- Нет схемы зала на Windows-терминале
- Слабый публичный API (по сравнению с iiko и Poster)
- Отказы/удаления блюд не задокументированы в API
- KDS-тайминги недоступны для внешних систем
- Офлайн: несколько терминалов не синхронизируются между собой
- Нет OLAP-аналитики (только стандартные отчёты)
